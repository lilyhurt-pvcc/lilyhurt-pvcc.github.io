Operating Systems Terminology

    Term Definition
    Operating System The software that controls the overall operation of a computer, providing the means by which a user can request the execution of programs, and provides the environment necessary to execute the programs requested.
    Job Queue In a batch processing system jobs (programs, data, and directions) wait for execution in FIFO fashion, unless priority is given.
    FIFO An abreviation of "first-in, first-out"
    Interactive Processing A feature of newer operating systems that allows for a program being executed to carry on a dialogue with the user through remote terminals.
    Time-Sharing Multiple users sharing access to a common computer.
    Multiprogramming The process of a given computer devoting a certain unit of time to multiple tasks, switching them every unit and then giving the illusion of multiple tasks being completed.
    Application Software Programs for performing tasks perticular to the machine's utilization.
    System Software Programs that perform tasks common to general computer systems.
    Utility Software Is system software not included by the OS, ie. formatting a disc, or compressing/decompressing data.
    Graphical User Interface (GUI) Objects to be manipulated (files, and programs) are represented pictorally on the display as icons.
    Shells Older user interfaces that communicate with users through textual messages using a keyboard and monitor screen.
    Kernel The operating system's internals, mediated by the user interface, that perform basic functions required by the computer instillation.
    Device Driver A software unit that communicates with the controllers to carry out operationson the peripheral devices attached to the machine.
    File Manager A unit in the kernel of a computer whose job is to coordinate the use of the machine's mass storage facilities, by keeping record of all files and where they are stored and who has access to them.
    Memory manager Coordinates the use of the machine's main memory, by allotting space as programs are utilized and keeping track of when memory is freed up.
    Virtual Memory Is created by the memory manager paging, or shuffling files that exceed the memory capacity into mass storage to give the illusion of excess memory
    Boot Strapping Or booting is the process the computer runs when it is powered on and transfers the operating system from mass storage to main memory.
    ROM (read-only memory) non-volitile memory meant to keep so that a computer has instructions when it first turns on to load the operating system (boot loader).
    Deadlock The condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another.
    Spooling holding data for output at a later but more convenient time to avoid deadlock.
    Super User (or Admisnistrator) A user with highly privledged access to the operating system and able to modify critical software packages, alter settings in the OS, and perform maintanace activities.
    Multi-Factor Authentication Requiring more than one peice of evidence to gain access to a system, ie. a password and a SMS code.
